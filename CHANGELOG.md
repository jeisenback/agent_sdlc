# Changelog

All notable changes to this project will be documented in this file.

Format: Keep a top-level `Unreleased` section and add entries. When cutting a
release, move entries under a new heading `vX.Y.Z - YYYY-MM-DD`.

Example
```
## Unreleased

- Added generator improvements

## v0.1.0 - 2026-03-14

- Initial public release
```
